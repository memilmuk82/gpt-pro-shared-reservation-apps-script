# ROADMAP_AND_BACKLOG.md

## 즉시 반영

### 1. 문서 갱신

사유:

- 현재 문서가 65 PASS / Phase 13 미구현 상태로 남아 있음
- 실제 기준은 73 PASS / Phase 13-A/B 완료
- 새 대화창에서 잘못 이어받는 문제 방지

### 2. Client_Backup.html 삭제

사유:

- 리팩토링 전 백업 파일
- 현재 include하지 않음
- Phase 13 이후 삭제 재판단 대상이었고 안정화 완료
- GitHub에 남기면 실제 운영 파일과 혼동됨

### 3. 일괄등록.csv 수동 배치

사유:

- Apps Script 운영 버전에서 예시 CSV 자동 다운로드 기능은 제외
- Drive 폴더에 수동 배치하는 방식이 단순하고 안전함

### 4. GitHub 보존

사유:

- 운영 코드와 개발 기록을 보존해야 함
- 리팩토링 전후 판단 근거와 테스트 결과를 남겨야 함

## 선택 반영

### GuideItems 변경 로그

반영 사유:

- 안내 문구와 보안 문구 변경 이력을 추적할 수 있음
- SettingsLog와 관리 일관성이 좋아짐

보류 가능 사유:

- 현재 GuideItems 수정 빈도가 낮으면 운영 차단 이슈가 아님
- GuideItemsLog 시트 추가가 필요할 수 있음
- 테스트 추가가 필요함

판단:

```text
운영자가 여러 명이고 안내 문구 변경이 자주 발생하면 구현.
그렇지 않으면 보류.
```

### 관리자 UI 미세 정렬

반영 사유:

- 관리자 화면 완성도 향상
- 포트폴리오 캡처/시연 품질 향상

보류 가능 사유:

- 기능 동작에는 영향 없음
- 이미 버튼 넘침 문제는 해결됨
- 운영 차단 이슈가 아님

판단:

```text
운영 전 최종 polish 단계에서 처리.
지금은 필수 아님.
```

### 사용 종료 Modal 체크리스트

반영 사유:

- 단순 문구보다 실제 행동 유도에 효과적
- 공용 PC에서 GPT/Google 로그아웃 확인을 명확히 요구할 수 있음

보류 가능 사유:

- 사용 종료 버튼, 수동 잠금, 자동 잠금, 로그아웃 안내가 이미 있음
- 기능 필수라기보다 UX 강화에 가까움

판단:

```text
공용 PC 사용 비율이 높으면 구현.
개인 업무용 PC 사용이 대부분이면 보류.
```

## 보류

### GuideItems 생성

보류 사유:

- 현재 기본 seed 항목 수정만으로 운영 가능
- 생성 기능은 guide_id, sort_order, active 기본값, 검증, UI, 테스트가 추가로 필요함
- 별도 승인 전까지 GuideItems 생성/삭제는 구현하지 않는 정책 유지

### GuideItems 삭제

보류 사유:

- 삭제는 복구가 어려움
- 보안 안내 항목을 실수로 삭제할 위험이 있음
- 현재 active=false로 숨김 처리 가능
- 필요하면 물리 삭제보다 soft delete가 적절하지만 시트 구조 변경이 필요할 수 있음

### 자동 잠금 시간 Settings 연동

보류 사유:

- 현재 30분 자동 잠금이 이미 구현됨
- Settings key 추가가 필요할 수 있음
- 숫자 범위 검증, seed, 테스트, 클라이언트 반영이 필요함
- 정책 변경에 가까움

### 네임스페이스 리팩토링

보류 사유:

- 현재 Client 파일 분리 구조로 충분히 안정적
- Apps Script HTMLService는 include 순서와 전역 함수 의존성이 중요함
- 리팩토링 이익보다 회귀 위험이 큼

## Flask + PostgreSQL + OCI 버전 백로그

Apps Script 운영 버전에서 제외한 다음 기능은 차기 포트폴리오 버전으로 이관한다.

```text
CSV 예시 파일 자동 생성
CSV 예시 다운로드 버튼
CSV 템플릿 버전 관리
CSV 업로드 이력 관리
검증 실패 로그 저장
관리자 다운로드 라우트
Google OAuth 로그인
PostgreSQL 기반 데이터 모델링
OCI 배포
```

차기 버전에서 이 기능들이 적합한 이유:

```text
Flask는 정적 파일 제공과 다운로드 라우트 구현이 자연스럽다.
PostgreSQL은 업로드 이력, 검증 로그, 템플릿 버전 관리에 적합하다.
OCI는 서버/DB/파일 제공 구조를 독립적으로 운영할 수 있다.
```
